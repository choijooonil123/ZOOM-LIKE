# ✅ 회의 참가 기능 확인 체크리스트

## 🔍 확인 사항

### 1. 백엔드 서버 상태 확인

**Render 백엔드 URL**: `https://zoom-like.onrender.com`

확인 방법:
1. 브라우저에서 직접 접속: `https://zoom-like.onrender.com/health`
2. 응답 확인:
   ```json
   {"status":"ok","timestamp":"..."}
   ```
3. 만약 응답이 없으면:
   - Render 대시보드에서 서비스 상태 확인
   - 슬리프 모드일 수 있음 (무료 플랜, 15분 비활성 시 잠듦)
   - 첫 요청 시 30초~1분 소요 가능

### 2. 프론트엔드-백엔드 연결 확인

**브라우저 개발자 도구(F12) → Console 탭**에서 확인:

#### ✅ 정상 작동 시:
```
서버에 연결되었습니다: [socket-id]
연결 확인: {sid: "..."}
```

#### ❌ 문제 발생 시:
- `Failed to fetch` → 백엔드 서버가 응답하지 않음
- `CORS error` → CORS 설정 문제
- `WebSocket connection failed` → Socket.io 연결 실패

### 3. 회의 참가 플로우 확인

#### 단계별 확인:

1. **로비 화면**
   - 표시 이름 입력
   - 방 ID 입력 또는 "새 회의실 생성"
   - "회의 참가" 버튼 클릭

2. **카메라/마이크 권한**
   - 브라우저에서 권한 요청 팝업 확인
   - "허용" 클릭

3. **회의 화면 전환**
   - 로비 화면이 숨겨지고 회의 화면이 표시됨
   - 로컬 비디오가 표시됨

4. **Socket.io 연결**
   - Console에서 "서버에 연결되었습니다" 메시지 확인
   - "사용자 [이름] ([sid])가 방 [room_id]에 참가했습니다" 메시지 확인

5. **다른 사용자 참가 테스트**
   - 다른 브라우저/기기에서 같은 방 ID로 접속
   - 서로의 비디오가 보이는지 확인

### 4. 네트워크 탭 확인

**브라우저 개발자 도구(F12) → Network 탭**:

1. **Socket.io 연결**
   - `socket.io/?EIO=...&transport=websocket` 요청 확인
   - Status: `101 Switching Protocols` (WebSocket 연결 성공)

2. **API 요청**
   - `/api/register`, `/api/login` 등이 `https://zoom-like.onrender.com`으로 전송되는지 확인

### 5. 일반적인 문제 해결

#### 문제: 백엔드 서버가 응답하지 않음
- **원인**: Render 슬리프 모드
- **해결**: 첫 요청 시 30초~1분 기다리기, 또는 UptimeRobot 등으로 주기적 ping

#### 문제: Socket.io 연결 실패
- **원인**: CORS 설정 또는 백엔드 URL 오류
- **해결**: 
  - `static/app.js`의 `BACKEND_URL` 확인
  - `server.py`의 CORS 설정 확인 (`allow_origins=["*"]`)

#### 문제: 회의 참가 후 아무것도 안 보임
- **원인**: 카메라/마이크 권한 미허용
- **해결**: 브라우저 설정에서 권한 허용

#### 문제: 다른 사용자가 보이지 않음
- **원인**: WebRTC 연결 실패
- **해결**: 
  - STUN 서버 설정 확인 (`app.js`의 `iceServers`)
  - 방화벽/네트워크 설정 확인

## 🧪 테스트 시나리오

### 시나리오 1: 단일 사용자 테스트
1. 브라우저에서 접속
2. 게스트 모드로 로비 진입
3. 새 회의실 생성
4. 회의 참가
5. 로컬 비디오가 보이는지 확인

### 시나리오 2: 다중 사용자 테스트
1. PC 브라우저에서 접속 → 방 생성
2. 휴대폰 브라우저에서 접속 → 같은 방 ID 입력
3. 서로의 비디오가 보이는지 확인
4. 채팅 메시지 전송 테스트

### 시나리오 3: 재접속 테스트
1. 방에 참가
2. 방 나가기
3. 로비로 돌아감
4. 방 ID가 자동으로 채워져 있는지 확인
5. 다시 참가 → 같은 방에 접속되는지 확인
